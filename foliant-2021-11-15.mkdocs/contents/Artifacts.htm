Artifacts

An artifact is either created or used by a team working on application development. For example, artifacts created by the Maven tool for application development include: JARs (Java ARchive), source and binary distributions, WARs (Web ARchive or Web Application Resource, etc.). Each maven artifact is identified by a group id, an artifact id, a version, an extension, and a classifier (the extension + classifier may be named by the type of artifact).

Click the Development menu item on the left and then the +Add new button at the top right of the Artifacts tab to add a new artifact or the edit icon in an existing artifact card to check and/or configure artifact parameters.

 

 

The Update artifact window appears.

 

 

Configure the following artifact parameters here:

  • Select a repository from the Repository drop-down menu.
  • Specify an artifact name in the Name field.
  • Repository Name
  • Group
  • Select Artifact type from the drop-down menu (Maven artifact, Docker container, NuGet package, etc.).
  • The Artifact field specifies the first part of the artifact name.
  • Select Extension type from the drop-down menu (war, jar, ejb, etc.)

Click the Update button to save changes.
