Handling security issues and defects

When the security pipeline scan is over, the results of the performed security tests are imported from AST tools into Maverix. They should be analyzed and processed. This is a task for Security Engineer.

Results of the security scan are collected, triaged, and analyzed by Maverix. Based on the machine learning model, Maverix performs automated analysis of security issues in the context of true/false positives. The same status, recommendations, assumptions about causes, and corrective actions should be assigned to similar true positive results. Maverix groups detected security issues based on the correlation model. As a result, similar true positive issues receive the same status and recommendations for remediation.

In this process, security issues found by AST tools are aggregated into security defects. Multiple security issues can be grouped into one security defect. Security defects are assigned priority and severity values. The security defect is exported from Maverix into the Defect Management System for further processing by the development team.

The workspace and project for a particular defect are defined based on the application settings and defect management configuration performed in Maverix in the earlier stages, for more details, see the Sync with Jira section. When the defect is fixed, Maverix analyzes the code changes and scan results based on pipeline data and confirms the new status. Also, Maverix analyzes the scan history and validates that security issues have been fixed properly.

Maverix provides an intuitive interface that allows users to view and group security issues, as well as create, filter, and verify security defects, and then synchronize security defects with the Defect Management System.
