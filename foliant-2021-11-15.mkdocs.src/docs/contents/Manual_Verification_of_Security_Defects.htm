Manual Verification of Security Defects

Every security defect shall be fixed by the application development team. When the defect is fixed, developers change the security defect status in Jira to "fixed". The defect now needs to be synchronized between Jira and Maverix.

As described above, select an application and click the Defects menu item on the left. The application defects page appears. Click the Sync with tracker button at the top right.

 

 

Wait for the synchronization to complete successfully. The security defect status in Maverix should now be "Fixed", just like in Jira.

To verify this security defect, Security Engineer runs a security scan, and verifies that the security defect is really fixed (i.e., there’s no corresponding security issue(s) in the latest security report from AST tool), and changes the status of this defect to "closed" in Maverix. After synchronizing defects between Maverix and Jira, this security defect gets the "Closed" status in Jira as well.
