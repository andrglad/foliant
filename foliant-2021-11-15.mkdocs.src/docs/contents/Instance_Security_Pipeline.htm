Instance Security Pipeline

Application instances are important from an application development perspective. Application instances are targeted for test stands, staging and production environments. Automated tests can be run for deployed application instances. Maverix supports DevSecOps pipelines of application instances.

Select the application on the Applications page and click the Development menu item on the left. Cards of application instances are located on the Instances tab of the application development page.

 

 

There can be multiple instances of an application for different stages (System test, Stage, Production, Integration Acceptance Test, User Acceptance Test) and different configurations for the same development stage.

To create an instance security pipeline, select the DevSecOps menu item on the left and then click the +Add new button at the top right of the Instance pipelines tab. Double-click the name of the application instance you want to add in the Choose the App Instance for pipeline window that appears.

 

 

The new instance security pipeline card appears on the Instance pipelines tab.

 

 

Click the Show pipeline config icon on the pipeline card. The pipeline details page appears. This page contains two warnings because the orchestration and scan tools haven’t been added to the pipeline yet. Click the Actions  button at the top right and select Add new element from the drop-down menu. Select the CI item in the window that appears to add the TeamCity orchestration tool to the pipeline, and then enter the tool parameters. Click the Actions  button at the top right and select the Add new element item from the drop-down menu. Select the DAST tool in the window that appears to add the scan tool to the pipeline and then enter the parameters for the selected tool.

In the Create DAST scan config window that appears, select the tool in the Tool field.

 

 

After selecting the Netsparker tool, the following required fields will appear in this window:

  • Specify the license key in the License field.
  • Select the Scan Profile.

If necessary, check the Include "Accepted Risk" and "False Positive" vulnerabilities item.

 

 

Click the DevSecOps menu item on the left and the edit button in the right bottom corner of the instance security pipeline card on the Instances tab to get the instance pipeline details. The instance pipeline details page appears.

 

 

A typical instance security pipeline contains three components (represented on the screen as three cards):

  • Application instance.
  • Orchestration tool.
  • AST tool.

The instance card represents the instance to be scanned for security issues. To configure the instance parameters, click the edit button on the instance card. The instance update window appears. See the "Application Instances" section for a description of the configuration of the instance parameters.

The orchestration tool card represents the corresponding tool for this pipeline (TeamCity). This card allows you to configure the orchestration tool parameters using the edit button on the tool card, including the node label, as in the case of artifact pipelines.

The AST tool card represents the AST tool chosen for this pipeline. For example, Netsparker is a DAST (Dynamic Application Security Testing) tool. Multiple AST tools can be selected for a single pipeline. Click the edit button on the AST tool card to tune the scanning parameters in the Update DAST scan config window.

When all the DevSecOps instance pipeline components are configured, click the Actions  button at the top right of the pipeline details page. Choose the Export CI/CD item from the drop-down menu to export the pipeline to the CI/CD orchestration tool.

After the successful export to CI/CD, click the Actions  button and select the Start scan item from the drop-down menu to start the instance security scan. Click the Start scan button in the appeared window. The start scan confirmation message appears an the bottom right of the Maverix screen.

Select the Scan history tab to check the status of the performed scans.
