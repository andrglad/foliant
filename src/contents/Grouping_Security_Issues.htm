Grouping Security Issues

Maverix finds the correlation between security issues imported from AST tools and groups them together. It performs an automated analysis of security issues in the context of true/false positives based on the internal machine learning model. The same status, recommendations, assumptions about causes, and corrective actions are assigned to similar true positive results. Based on the correlation model, the identified security issues are grouped and aggregated into security defects.

Also, security issues can be grouped manually. Click the Show filters button at the top right to setup the filter. There are four filtering modes: one using correlation rules and three customizable.

Switch on the Apply correlation rules by default selector and select a correlation rule to enable filtering using the correlation rule (only for SAST vulnerabilities).

 

 

Three customizable filters offer the following options:

  • By Issue properties
  • By Linked entities
  • By AVC.

By choosing the Issue properties filtering option, we get the ability to sort security issues by the following criteria:

  • Sort by — Severity: High to Low/Low to High and Newest first/Last updated.
  • By type (SAST, SCA Compliance, SCA Security, and DAST).
  • By status (To Verify, Confirmed, Open, Fixed, and False Positive).
  • By tool (NetSparker, Acunetix, MDast, Wallarm, IBM Security AppScan, Fortify Static Code Analyzer, and SonarQube).
  • By severity (Low, Medium, High, and Critical).
  • By source.
  • By category.
  • By CWE ID.
  • Updated in (Last week, Last month, Last 3 months, Last 6 months, Last year, Last 2 years, and Last 3 years).

Select the Linked entities filtering option to sort security issues based on the associated entities (codebases, artifacts, or instances) using the following fields:

  • Sort by — Severity: High to Low/Low to High and Newest first/Last updated.
  • By type (SAST, SCA Compliance, SCA Security, and DAST).
  • By codebase
  • By artifact
  • By instance
  • By repository proxy

 

Several parameters can be selected simultaneously in the by type, by codebase, by artifact, by instance, and by repository proxy fields. Please see picture below..


 

Select the AVC filtering option to sort out security issues using the AVC analysis results:

  • Sort by — Severity: High to Low/Low to High and Newest first/Last updated.
  • By type (SAST, SCA Compliance, SCA Security, and DAST).
  • By AVC status (False Positive, True Positive).
  • By lower limit of accuracy.
  • By upper limit of accuracy.

For the AVC option for each issue, the AVC status is displayed in the corresponding column. It indicates whether the issue is True Positive (TP) or False Positive (FP) as assessed by Maverix. In addition, the probability of the assessment provided by Maverix for the issue is displayed next to the issue status.
