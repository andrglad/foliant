Application Settings

Each application needs to be configured before team members can work on it in Maverix. Application configuration in Maverix covers several areas.

Click the Show App details icon to open the application page.

 

 

The configuration process involves defining and setting application parameters using menu items on the left of the application page.

 

 

  • The Info page contains information about the application. This page provides the current status of the application in terms of scan, issue, and defect metrics, see picture above. It also shows the application codebase size in SLOC, risk profile, issues summary, defects summary, and SSDL summary. All this data is retrieved from the data warehouse.
  • The Development page contains five tabs (see the "Application configuration" section).
  • The DevSecOps page is intended for working with application security pipelines (see the "Security Pipelines" section).
  • The Issues and Defects pages are intended for working with application security issues and defects. For more information, see the "Handling of security issues and defects" section.
  • The Defect tracking page is used to configure security defect tracking (for more information, see the "Sync with Jira" section).
  • The SSDL page contains information related to the Secure Software Development Lifecycle.
  • The Settings page contains application settings.
  • The Notifications page contains settings related to the application notifications. For more information, see the "Application notifications" section.
  • The Risk Acceptance page contains information on security issues for which risk acceptance rules are created, as well as a list of active Risk Acceptance rules. For more information, see the "Risk acceptance" section.
  • The Task log page contains a list and status of tasks performed on the back-end for this application. For more information, see the "Task Log" section).
  • The steps for setting up the application using the Settings page are described below.

To get more detailed information about the application, it’s necessary to go to the Applications page and click the Show app details icon.

 

 

The application page with detailed information about the application appears on the screen. Click the Settings menu item on the left to edit the information and configure the application. The application settings page with five tabs appears.

 

 

The General tab allows you to configure important application settings:

  • The Application name is the name of the application in the system.
  • The Code is the internal code of the application in external user systems (this field can be left blank).
  • The EXTERNAL ID is the external identifier of the application. This parameter is passed by external tools (e.g., build UUID in Teamcity).
  • The Description is a short description of the application.
  • and perform the other settings described below.
